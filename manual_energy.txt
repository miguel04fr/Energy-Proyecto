MANUAL TÉCNICO Y DE USUARIO - APLICACIÓN ENERGY
=============================================

1. INTRODUCCIÓN
--------------
Energy es una aplicación web para la gestión de actividades deportivas que permite a los usuarios descubrir, inscribirse y gestionar su participación en diferentes deportes. La aplicación está desarrollada en Java EE con JSP y utiliza una base de datos MySQL.

2. REQUISITOS TÉCNICOS
---------------------
- Java JDK 17 o superior
- Servidor de aplicaciones compatible con Java EE 7.0
- MySQL 8.0 o superior
- Navegador web moderno (Chrome, Firefox, Safari, Edge)

3. ESTRUCTURA DE LA APLICACIÓN
----------------------------
La aplicación sigue una arquitectura MVC (Modelo-Vista-Controlador):

3.1 Modelo
---------
- Beans: Deporte, Entrenador, EntrenadorDeporte, Horario, Inscripcion, Usuario
- DAOs: Interfaces y implementaciones para cada entidad
- DAOFactory: Patrón Factory para la creación de DAOs

3.2 Controladores
---------------
- FrontController: Controlador principal que maneja todas las peticiones
- Login: Gestión de autenticación
- Create: Creación de registros
- Delete: Eliminación de registros
- HorarioAjaxController: Gestión de horarios mediante AJAX
- RegistroDeporte: Gestión de registros deportivos

3.2.1 Gestión de Atributos
------------------------
La aplicación utiliza atributos de request para pasar datos entre controladores y vistas:

Ejemplo de uso:
```java
request.setAttribute("listaDeportes", listaDeportes);
```

Este atributo puede ser accedido en las vistas JSP usando:
```jsp
${listaDeportes}
```

Los deportes disponibles en el sistema son:
- Fútbol
- Baloncesto
- Tenis
- Natación
- Crossfit
- Pádel
- Voleibol
- Atletismo
- Badminton
- Gimnasio

3.3 Vistas
---------
- JSPs para la interfaz de usuario
- CSS para estilos
- JavaScript para interactividad
- Componentes reutilizables en la carpeta INC

4. MANUAL DE USUARIO
------------------

4.1 Acceso a la Aplicación
------------------------
- Abrir el navegador web
- Ingresar la URL de la aplicación
- La página principal muestra un banner con información general
- Opciones de login/registro en la barra de navegación

4.2 Navegación Principal
----------------------
- Menú superior con opciones de navegación
- Carrusel de deportes disponibles
- Secciones de información y servicios
- Pie de página con información de contacto

4.3 Funcionalidades Principales
----------------------------

4.3.1 Registro de Usuarios
------------------------
- Clic en "Registrarse" en la barra de navegación
- Completar formulario con datos personales
- Verificar correo electrónico
- Confirmar registro

4.3.2 Inicio de Sesión
--------------------
- Clic en "Iniciar Sesión" en la barra de navegación
- Ingresar credenciales
- Acceder al panel de usuario

4.3.3 Gestión de Deportes
------------------------
- Explorar deportes disponibles en el carrusel
- Ver detalles de cada deporte
- Inscripción en actividades deportivas
- Consulta de horarios

4.3.4 Reservas
------------
- Seleccionar deporte
- Elegir horario disponible
- Confirmar reserva
- Gestionar reservas existentes

4.4 Características Adicionales
----------------------------
- Modo oscuro/claro
- Navegación responsiva
- Animaciones y transiciones
- Notificaciones del sistema

5. MANUAL TÉCNICO
---------------

5.1 Configuración del Entorno
---------------------------
1. Clonar el repositorio
2. Configurar la base de datos MySQL
3. Importar el proyecto en NetBeans
4. Configurar el servidor de aplicaciones
5. Ejecutar el script SQL de creación de tablas

5.2 Estructura de la Base de Datos
--------------------------------
- Tabla usuarios: Información de usuarios
- Tabla deportes: Catálogo de deportes
- Tabla entrenadores: Información de entrenadores
- Tabla horarios: Disponibilidad de actividades
- Tabla inscripciones: Registro de usuarios en actividades
- Tabla reservas_pista: Gestión de reservas

5.3 Dependencias
--------------
- Java EE 7.0
- MySQL Connector Java 8.0.26
- JSON 20231013
- JSTL 1.2
- Commons BeanUtils 1.8.3
- Gson 2.8.9

5.4 Configuración de Seguridad
---------------------------
- Filtro UTF8 para codificación de caracteres
- Validación de formularios
- Control de sesiones
- Protección contra inyección SQL
